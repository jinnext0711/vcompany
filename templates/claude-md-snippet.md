# CLAUDE.md スニペット：スライド・資料生成の品質ルール

> このファイルの内容をプロジェクトの CLAUDE.md に追加してください。
> Claude Codeはセッション開始時にCLAUDE.mdを読み込むため、
> プロンプトで毎回指示しなくても品質ルールが自動的に適用されます。

---

## CLAUDE.md に追加する内容

```markdown
# スライド・資料生成ルール

## 絶対禁止（違反は全体やり直し）
- タイトル下にアクセントライン（装飾的な水平線）を入れない
- 装飾的な円・楕円・幾何学模様を意味なく配置しない
- ◎ ○ △ × の記号をテーブルのセルに使わない（テキスト評価+色分けで代替）
- Unicode bullet "•" を直接テキストに書かない（PptxGenJSの bullet: true を使う）
- スライドの下半分（y > 3.5 inch）を空白にしない
- 全セル格子罫線のテーブルを作らない（横罫線のみ）
- PptxGenJSの色コードに # を含めない（ファイル破損する）
- PptxGenJSのオプションオブジェクトを複数の要素間で再利用しない（破壊的変更される）

## レイアウト基準
- 全スライドの最下要素の (y + h) が 4.5 inch 以上であること
- チャートの h は 3.5 inch 以上
- テーブルの rowH の合計は 3.5 inch 以上
- テキストボックスの h はコンテンツ量に比例させる（3行で h=2.5 は過大）

## テーブル品質
- 罫線: border: { type: "none" } をデフォルトにし、個別セルで bottom のみ指定
- 数値: 右揃え、3桁カンマ区切り、正の増減率に + を付ける
- ヘッダー: 背景色あり（1E2761）、白文字、Bold
- 合計行: 背景色あり（E8EBF0）、上辺に太い罫線
- 比較表の評価: 色分けテキスト（緑: 0D7C3D / 黒: 333333 / 橙: F59E0B / 赤: C62828）

## QAフロー
1. コード生成 → 実行
2. LibreOfficeでPDF変換 → pdftoppmで画像化 → markitdownでテキスト抽出
3. QAサブエージェント（qa-agent-prompt.md）で検査 → qa-report.md を出力
   - 自分自身（コード生成者）がQAすることは禁止。確認バイアスにより問題を見逃す
4. qa-report.md の Critical / Major を全て修正 → 再実行 → 再検査
5. 全 Critical / Major がゼロになったらユーザーに画像 + qa-report.md を提示
```

---

## 使い方

プロジェクトのルートに `CLAUDE.md` がない場合は新規作成、
既にある場合は末尾に上記を追加してください。

```bash
# 新規作成の場合
cat >> CLAUDE.md << 'EOF'
（上記の内容をペースト）
EOF

# またはClaude Codeで直接
claude "CLAUDE.md にスライド生成の品質ルールを追加して。ルールは以下: ..."
```

CLAUDE.md に書いたルールは、そのプロジェクト内の全セッションで自動的に適用されます。
プロンプトで毎回「◎○△×を使うな」と書く必要がなくなります。
