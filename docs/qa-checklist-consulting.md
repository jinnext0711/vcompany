# コンサルティングファーム品質 資料QAチェックリスト

本チェックリストは、コンサルティングファームが求める品質水準で資料をレビューするためのものです。
各項目について「合格基準」と「よくあるNG例」を確認し、すべての項目をクリアした状態で納品してください。

---

## 1. 論理構造チェック

### 1-1. ピラミッド原則

| 観点 | 内容 |
|------|------|
| **合格基準** | 結論が最上位に配置され、その下に根拠が階層的にぶら下がっている。各階層で「なぜそう言えるのか？」に答える構造になっている。帰納法または演繹法のいずれかで一貫している。 |
| **よくあるNG例** | 時系列順に並べただけの報告書形式。結論が最後に出てくる「起承転結」型。根拠の階層が混在し、抽象度のレベルが揃っていない。 |

### 1-2. SCQAフレームワーク

| 観点 | 内容 |
|------|------|
| **合格基準** | Situation（状況）→ Complication（課題）→ Question（問い）→ Answer（答え）の流れが資料冒頭で明確に提示されている。読み手がなぜこの資料を読む必要があるのかが最初の数スライドで理解できる。 |
| **よくあるNG例** | いきなり分析結果から始まり、前提や問題設定が不明。SituationとComplicationが曖昧で、何が課題なのかが分からない。Questionが暗黙のままでAnswerとの対応が取れていない。 |

### 1-3. ストーリーライン（アクションタイトル通読テスト）

| 観点 | 内容 |
|------|------|
| **合格基準** | 各スライドのアクションタイトルだけを順に読んだとき、資料全体のストーリーが一本の論理として通じる。タイトルの連なりがエグゼクティブサマリーの代替として機能する。 |
| **よくあるNG例** | タイトルが「市場分析」「競合比較」「財務データ」のようなトピック名の羅列になっており、主張が読み取れない。タイトル同士の論理的なつながりが希薄で、飛躍がある。 |

### 1-4. MECE（漏れなくダブりなく）

| 観点 | 内容 |
|------|------|
| **合格基準** | 各階層の分類が相互に排他的（ダブりなし）かつ全体を網羅（漏れなし）している。既知のフレームワーク（3C、4P、バリューチェーン等）を適切に活用して構造化されている。 |
| **よくあるNG例** | 分類項目間に重複がある（例：「営業改善」と「売上向上施策」が並列に存在）。重要な観点が抜け落ちている（例：競合分析で代替品の脅威が欠落）。分類の切り口が途中で変わっている。 |

## 2. スライドレベルチェック

### 2-1. アクションタイトル（So What）

| 観点 | 内容 |
|------|------|
| **合格基準** | タイトルが主張文（So What）になっている。読み手がタイトルだけ読んでもスライドの結論が分かる。具体的な数値や比較が含まれている場合はなお良い（例：「国内EC市場は年率15%で成長し、2027年に30兆円に到達する見込み」）。 |
| **よくあるNG例** | 「市場概況」「売上推移」「組織体制」のようなトピック名だけのタイトル。「分析結果について」のような曖昧な表現。主語と述語が欠けており、文として成立していない。 |

### 2-2. 1-1-1ルール

| 観点 | 内容 |
|------|------|
| **合格基準** | 1スライドに1つのメッセージのみが記載されている。そのメッセージを支持する根拠構造（チャート、表、箇条書き等）が1つの形式で統一されている。スライドを分割すべき内容が無理に詰め込まれていない。 |
| **よくあるNG例** | 1枚のスライドに複数の論点が混在している。メッセージと無関係な補足情報が同居している。支持構造がチャートと表と箇条書きで混在し、視線の流れが定まらない。 |

### 2-3. 10秒ルール

| 観点 | 内容 |
|------|------|
| **合格基準** | スライドを見て10秒以内に「何を言いたいスライドか」が把握できる。視線の流れが自然（左上→右下、またはZ型/F型）で迷わない。重要な情報が視覚的に強調されている。 |
| **よくあるNG例** | 文字量が多すぎて読み切れない。要素が乱雑に配置され、どこから見るべきか分からない。強調（太字・色・サイズ）が多すぎて優先度が判別できない。 |

### 2-4. スタンドアローン性

| 観点 | 内容 |
|------|------|
| **合格基準** | 口頭説明がなくても、スライド単体で内容が理解できる。グラフの軸ラベル・凡例・単位が明記されている。略語には初出時に正式名称が併記されている。出典が明記されている。 |
| **よくあるNG例** | 「ここは口頭で補足します」前提の空白スライド。グラフに軸ラベルや単位がない。「前述の通り」と書かれているが参照先が不明。データの出典・時点が記載されていない。 |

## 3. デザインチェック

### 3-1. 3層構造（ヘッダー / ボディ / フッター）

| 観点 | 内容 |
|------|------|
| **合格基準** | 全スライドで以下の3層が統一されている。ヘッダー：アクションタイトル（主張文）。ボディ：タイトルを支持する図表・テキスト。フッター：ページ番号、出典、機密区分等。各層の高さ・位置が全スライドで揃っている。 |
| **よくあるNG例** | タイトルの位置やフォントサイズがスライドごとに異なる。フッターにページ番号がないスライドが混在。ボディ領域にタイトル的な要素が侵入し、層の境界が曖昧。 |

### 3-2. 配色 70-25-5 ルール

| 観点 | 内容 |
|------|------|
| **合格基準** | ベースカラー70%（背景・本文）、メインカラー25%（見出し・枠線・図表の主要色）、アクセントカラー5%（強調・CTA）の比率が守られている。色数は全体で4色以内に抑えられている。 |
| **よくあるNG例** | 5色以上の色が無秩序に使われている。強調色が多用されて「すべてが強調」状態になっている。背景色とテキスト色のコントラスト比が不足し、可読性が低い。 |

### 3-3. フォントの統一性

| 観点 | 内容 |
|------|------|
| **合格基準** | 使用フォントは最大2種類（和文1 + 欧文1）に統一。フォントサイズの階層が明確（例：タイトル24pt、サブタイトル18pt、本文14pt、注釈10pt）。太字・斜体の使い方に一貫したルールがある。 |
| **よくあるNG例** | 3種類以上のフォントが混在している。コピー＆ペーストにより他資料のフォントが混入。フォントサイズが場当たり的に変えられており、階層感がない。 |

### 3-4. 余白・マージンの一貫性

| 観点 | 内容 |
|------|------|
| **合格基準** | スライド四辺のマージンが統一されている（推奨：上下左右とも15mm以上）。要素間のスペーシングが均等。整列（左揃え・中央揃え等）が一貫している。グリッドまたはガイドラインに沿って配置されている。 |
| **よくあるNG例** | 要素がスライドの端まで詰め込まれ、余白がない。要素の配置が微妙にずれている（1〜2pxレベルの不揃い）。余白の広さがスライドごとに異なる。 |

### 3-5. 図表の品質

| 観点 | 内容 |
|------|------|
| **合格基準** | チャートジャンク（装飾的な3D効果、不要なグリッド線、過剰な装飾）が排除されている。データインク比（データを表現するインクの割合）が高い。グラフの種類が伝えたいメッセージに適している（比較→棒グラフ、推移→折れ線、構成比→積み上げ棒 or ウォーターフォール）。 |
| **よくあるNG例** | 3D円グラフが使われている（視覚的に歪み、正確な比較ができない）。グリッド線が濃く、データより目立っている。不必要な装飾画像やイラストがチャート内に配置されている。軸の起点がゼロでないのに注記がない。 |

## 4. テキストチェック

### 4-1. 3点ルール

| 観点 | 内容 |
|------|------|
| **合格基準** | 箇条書きは最大3項目に集約されている。4つ以上ある場合はグルーピングして上位概念でまとめ直している。各項目の粒度（抽象度）が揃っている。並列する項目の文体が統一されている（体言止め or 用言止め）。 |
| **よくあるNG例** | 箇条書きが5〜7項目並んでおり、読み手が記憶できない。項目の粒度がバラバラ（「DX推進」と「Excelマクロの修正」が並列）。体言止めと文末表現が混在している。 |

### 4-2. 冗長な表現の排除

| 観点 | 内容 |
|------|------|
| **合格基準** | 「〜することが可能である」→「〜できる」のように簡潔に書き換えられている。二重否定が使われていない。一文が60文字以内に収まっている。修飾語が最小限に抑えられている。 |
| **よくあるNG例** | 「〜という点に関しましては」「〜であるということが考えられる」等の回りくどい表現。同じ意味の言葉の繰り返し（「まず最初に」「各々それぞれ」）。主語と述語の距離が遠く、文意が取りにくい。 |

### 4-3. 専門用語の適切な使用

| 観点 | 内容 |
|------|------|
| **合格基準** | 読み手のリテラシーレベルに応じた用語選択がされている。初出の専門用語・略語には定義または注釈が付与されている。社内用語とクライアント用語が適切に使い分けられている。同一概念に対して資料内で用語が統一されている。 |
| **よくあるNG例** | クライアントが知らない業界用語が注釈なしで使われている。同じ概念に対して「KPI」「指標」「メトリクス」が混在。カタカナ用語の乱用（「アジャイルにピボットしてスケールする」等）。 |

## 5. ボリュームチェック

### 5-1. テーマに対する深さ

| 観点 | 内容 |
|------|------|
| **合格基準** | 各テーマに対して「現状分析 → 示唆 → 提言」の3段階が揃っている。定量データ（ファクト）と定性分析（インサイト）の両方が含まれている。「だから何？（So What）」「本当にそうか？（True？）」「それだけか？（Enough？）」の3つの問いに耐えうる深さがある。 |
| **よくあるNG例** | データの羅列だけで示唆・提言がない（分析が浅い）。提言はあるが裏付けとなるデータがない（根拠が弱い）。表面的な一般論に留まり、クライアント固有の洞察がない。 |

### 5-2. セクション間のバランス

| 観点 | 内容 |
|------|------|
| **合格基準** | 重要度の高いセクションにより多くのスライドが割かれている。各セクションの深さが均等（あるセクションだけ極端に薄い・厚いということがない）。全体のページ数が目的・用途に対して適切（経営会議向け：10-15枚、詳細報告：30-50枚が目安）。 |
| **よくあるNG例** | 現状分析に大半のページを割き、肝心の提言が1枚だけ。重要度の低いセクションが過剰に詳細で、資料全体の焦点がぼやけている。Appendixに本来ボディに入れるべき重要情報が押し込まれている。 |

## QAレビュー実施手順

1. **セルフチェック**：作成者自身が本チェックリストで全項目を確認する
2. **ピアレビュー**：同僚が「10秒ルール」と「ストーリーライン通読テスト」を実施する
3. **マネージャーレビュー**：論理構造とメッセージの妥当性を確認する
4. **最終QA**：品質管理部がデザイン・テキスト・ボリュームの全項目を確認する
