# vCompany デザインルール — コンサルティング品質基準

## 基本原則

- **アクションタイトル**: 全スライドのタイトルは「だから何なのか（So What）」を完結した主張文で記載する
- **1-1-1ルール**: 1スライド → 1メッセージ → 1つの支持構造（図表 or 箇条書き）
- **スタンドアローン性**: タイトルだけ読めばストーリー全体が分かること
- **10秒ルール**: 各スライドは10秒以内に理解できること

## スライド3層構造

```
┌─────────────────────────────────────────────────┐
│ ヘッダー: アクションタイトル（主張文）1-2行        │ ← 最重要
├─────────────────────────────────────────────────┤
│                                                 │
│  ボディ: 主張を裏付ける根拠                       │
│  （図表・チャート・構造化テキスト）                 │ ← スライドの80%
│                                                 │
├─────────────────────────────────────────────────┤
│ フッター: 出典・ページ番号                         │
└─────────────────────────────────────────────────┘
```

## アクションタイトルの書き方

### 悪い例（トピック名）
- 「市場規模」「主な機能」「料金体系」

### 良い例（主張文）
- 「インドのOTC鎮痛薬市場は2030年までに2倍に成長」
- 「5つのコア機能が開発ワークフロー全体をカバーする」
- 「Pro $20/月から利用可能、ヘビーユーザーにはMax推奨」

## 配色ルール（70-25-5）

| 役割 | 比率 | カラー | 用途 |
|------|------|--------|------|
| ベース | 70% | 白 `#FFFFFF` / ライトグレー `#F5F5F5` | 背景、余白 |
| メイン | 25% | 濃紺 `#1A1A2E` | テキスト、見出し、主要データ |
| アクセント | 5% | シアン `#53868B` | ハイライト、重要インサイト |

- 赤は問題・警告のみに使用
- 同色の明度変化でメリハリを出す

## フォント基準（コンサル準拠）

| 要素 | サイズ | ウェイト |
|------|-------|---------|
| アクションタイトル | 20-22pt | Bold |
| セクション見出し | 16-18pt | Bold |
| 本文・箇条書き | 14pt | Regular |
| データラベル・注釈 | 11-12pt | Regular |
| フッター・出典 | 9-10pt | Regular |

- フォントは Noto Sans JP の1種類のみ
- 強調は **太字のみ**（イタリック・下線禁止）

## テキストルール

- **3点ルール**: 1スライドの箇条書きは最大3つ
- 太字キーワード + 補足説明の構造
- 本文テキストは左揃え
- 行間 1.3-1.5

## 図表の使い分け

| データの性質 | 推奨図表 |
|------------|---------|
| 比較 | テーブル、棒グラフ |
| 推移 | 折れ線グラフ |
| 構成比 | 横棒100%、円グラフ（5項目以下） |
| 増減分析 | ウォーターフォールチャート |
| 分類・優先度 | 2x2マトリクス |
| プロセス | フロー図（横方向） |
| 評価 | Harvey Ball、信号機インジケーター |
| 論理構造 | ピラミッド図 |

## スライドレイアウトパターン

1. **タイトル** — 資料タイトル + サブタイトル
2. **エグゼクティブサマリー** — 3-5つの主要メッセージを構造化
3. **アクションタイトル + 図表** — コンサルの標準スライド
4. **アクションタイトル + テーブル** — 比較・一覧
5. **アクションタイトル + KPI** — 数値ハイライト
6. **アクションタイトル + 2x2マトリクス** — ポジショニング
7. **アクションタイトル + フロー図** — プロセス説明
8. **アクションタイトル + 構造化テキスト** — 太字KW+補足（箇条書き不可避時）
9. **セクション区切り** — セクション名のみ
10. **まとめ + Next Steps** — キーメッセージ + 次のアクション
