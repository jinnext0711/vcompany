# vCompany 組織構成書

## 組織図

```
                      ┌──────────────┐
                      │   Owner      │
                      │ (ユーザー)    │
                      └──────┬───────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
   ┌──────┴───────┐   ┌─────┴──────┐   ┌───────┴──────┐
   │   Auditor    │   │    CEO     │   │  Secretary   │
   │  (監査役)     │   │  (統括)    │   │   (秘書)     │
   └──────────────┘   └─────┬──────┘   └──────────────┘
                            │
 ┌─────┬─────┬─────┬────┬───┼───┬─────┬─────┬─────┐
 │     │     │     │        │   │     │     │     │
企画  開発  品質  広報  法務  総務  経理  Data  人事
Plan  Dev   QA   PR  Legal  GA  Fin   Ana   HR
      └┬─┘
  Front Back Infra
```

## 部門一覧

| # | 部門 | エージェント | 報告先 | 主な役割 |
|---|------|------------|--------|---------|
| - | Owner | ユーザー | - | 最終意思決定、経営方針 |
| 1 | 監査役 | auditor | Owner（直接） | 組織全体のガバナンス監査 |
| 2 | CEO | ceo | Owner | 各部門への指示・統括 |
| 3 | 秘書 | secretary | CEO | MCP連携、タスク管理、スケジュール |
| 4 | 企画部 | planning | CEO | 要件定義、仕様策定、リサーチ |
| 5 | 開発部 | engineering | CEO | 実装統括 |
| 5a | └ フロントエンド | frontend | 開発部 | UI/UX実装 |
| 5b | └ バックエンド | backend | 開発部 | API・DB・ビジネスロジック |
| 5c | └ インフラ | infra | 開発部 | CI/CD・デプロイ・クラウド |
| 6 | 品質管理部 | qa | CEO | テスト、コードレビュー、品質基準 |
| 7 | 広報部 | pr | CEO | ドキュメント、リリースノート |
| 8 | 法務部 | legal | CEO | 著作権、ライセンス、コンプライアンス |
| 9 | 総務部 | general-affairs | CEO | プロジェクト設定、依存関係管理 |
| 10 | 経理部 | finance | CEO | API利用料、コスト管理、予算 |
| 11 | データ分析部 | analytics | CEO | KPI分析、ログ分析、データ基盤 |
| 12 | 人事部 | hr | CEO | エージェントのパフォーマンス管理・改善 |

## ワークフロー

```
1. Owner → CEO に方針・要望を伝える
2. CEO → Secretary に情報収集・タスク整理を依頼
3. CEO → 各部門に具体的な作業指示を出す
4. 各部門 → 成果物を CEO に報告
5. CEO → Owner に結果を報告
6. Auditor → 独立した立場で監査し、Owner に直接報告
```
