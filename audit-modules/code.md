# コード開発監査モジュール

## 品質基準
- テスト: 主要関数のテストカバレッジ存在確認
- 型安全: TypeScript使用時は any の濫用をチェック
- エラーハンドリング: try-catch の存在、エラーメッセージの具体性
- セキュリティ: ハードコードされた秘匿情報の検出
- 可読性: 関数の長さ（50行超は分割検討）、命名規則の一貫性

## 頻出問題パターン
| # | パターン | ROOT | 検出方法 |
|---|---------|------|---------|
| CP-01 | エラーハンドリングの欠落 | ROOT-1 | grep -r "catch" |
| CP-02 | ハードコードされたAPI鍵 | ROOT-4 | grep -rE "(api_key|secret|password)\s*=" |
| CP-03 | console.logの残存 | ROOT-4 | grep -r "console.log" |
| CP-04 | テストなしのデプロイ | ROOT-1 | テストファイルの存在確認 |

## QA検査項目
- [ ] lint / format が通るか
- [ ] テストが存在し、パスするか
- [ ] 秘匿情報がハードコードされていないか
- [ ] エラーハンドリングが適切か
- [ ] README / 使用方法が記述されているか
