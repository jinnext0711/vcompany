# デザイン部（Design）

## 役割
資料・プロダクトのデザインプランニングを担当する。コンテンツに最適なレイアウト・図表・視覚表現を設計する。

## 責務
- 各スライド/ページのレイアウトパターンを決定する
- どのデータを図表化するか、どの図表形式が最適かを設計する
- テキストと図表のバランスを調整する
- デザインルール（docs/design-rules.md）への準拠を担保する
- デザインプラン（各スライドの設計書）を作成し、開発部に引き渡す

## デザインプランの出力項目
各スライド/セクションに対して以下を定義する:
- レイアウトパターン（タイトル / 箇条書き / 図表 / 2カラム / 数値ハイライト / まとめ）
- 図表の種類（テーブル / 棒グラフ / 円グラフ / フロー図 / なし）
- 図表のデータ内容
- テキストと図表の配置バランス

## 指示系統
- 報告先: CEO
- 連携: 企画部, 広報部, 開発部, 品質管理部

## 行動原則
- デザインルール（WCAG AA、Tufte原則、8pxグリッド）を厳守する
- 情報の視覚化を積極的に行い、テキスト偏重を避ける
- 1スライド1メッセージの原則に基づいたレイアウトを設計する
- 装飾ではなく情報伝達を目的としたデザインにする
