# 資料作成機能 — プロジェクト共通設定

## 概要
社内外で使用するビジネス資料を作成する機能。
出力形式はpptx（プレゼンテーション）とhtml（Webレポート・ダッシュボード）。
対象読者は経営層、事業部門責任者、クライアント企業の担当者。

## 資料の用途分類

### 社内資料
- 目的: 意思決定の材料、情報共有、進捗報告
- トーン: 簡潔・直截。結論とアクションアイテムを最優先
- 機密表記: ヘッダーまたはフッターに「社外秘」「Confidential」を付与

### 社外資料
- 目的: 提案、報告、情報提供
- トーン: フォーマル。専門用語には補足説明を添える
- 機密表記: 全ページに「Confidential」を付与
- ロゴ・ブランドカラー: クライアント指定に従う
- データ更新日時を明示する

## デザイン基本設定

### 配色（社内外共通ベース）
- メインカラー: #2E5090
- アクセント: #E67E22
- テキスト: #333333
- 背景: #FFFFFF
- サブ背景: #F5F7FA
- 罫線・区切り: #E0E0E0
- 成功/正: #27AE60
- 警告/注意: #F39C12
- エラー/負: #E74C3C

### フォント
- pptx: Noto Sans JP（見出しBold）、本文14pt以上、タイトル24pt以上
- html: font-family: "Noto Sans JP", "Hiragino Sans", "Meiryo", sans-serif

## 出力ルール
- pptx出力先: ./output/pptx/
- html出力先: ./output/html/
- ファイル名規則: YYYYMMDD_資料タイトル_vX.X
- htmlは単一ファイルにまとめる（CSS・JSをインライン化）
- 画像はBase64エンコードでhtmlに埋め込む

## 作業フロー
1. 資料の種類（pptx or html）と用途（社内 or 社外）を確認する
2. 構成案（アウトライン）を作成し、承認を得る
3. 本文・ビジュアルを作成する
4. セルフレビュー（誤字脱字、数値整合性、出典確認）を実施する
5. **Phase 6.5 視覚品質検証を実行する**（pptxの場合）
6. Error 0件になるまで修正→再検証サイクルを繰り返す
7. 完成版を所定の出力先に保存する

## 品質保証の原則
- **自己評価と実物の乖離を許さない**: Grade Aを付けるなら、実際の出力ファイルを開いて確認済みであること
- **JSONの検証だけでは不十分**: レンダリング後の視覚的品質まで検証する（Phase 6.5）
- **コンテンツ品質とレンダリング品質は別**: 内容が良くても見た目が悪ければ使えない
- **空スライドは絶対に許容しない**: データの不一致やゾーン名のミスマッチを事前に検出する
