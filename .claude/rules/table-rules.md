# テーブル生成のルール v2

このルールはSTEP 2（ビジュアル実装）のテーブル生成に適用される。slide-master-plan.md の設計仕様に基づいて実装すること。

## 最重要: 絶対禁止（コード書く前に暗記 — 違反は全成果物やり直し）

1. ◎ ○ △ × の記号をセルに書くな → テキスト評価+色分けで代替せよ（違反は成果物やり直し）
2. 全セル格子罫線を使うな → border:{type:"none"} + bottom個別指定（違反は成果物やり直し）
3. 数値を左揃えにするな → 数値は必ず align:"right"（違反は不合格）

❌ `["機能", "◎", "○", "△"]` → 書いた時点でやり直し
✅ `["機能", {text:"完全自律", options:{color:"0D7C3D"}}, ...]`

## 出力形式

| 形式 | 用途 |
|------|------|
| pptx | プレゼンテーション内の埋め込みテーブル |
| html | Web掲載用、インタラクティブ表示 |
| xlsx | データ配布用、編集可能な形式 |
| svg | 印刷品質、ベクター形式での配布 |

## コンサル品質テーブルの実装ルール

### 罫線
- border: { type: "none" } をデフォルトにする
- 横罫線のみ: ヘッダー下（1.5pt）と合計行上（1pt）
- データ行の下: 薄い区切り線（0.5pt, E5E7EB）を bottom のみ指定
- 縦罫線は禁止（発見次第やり直し）

### テキスト配置
- 数値: 右揃え（align: "right"）
- テキスト: 左揃え（align: "left"）
- ヘッダー: 背景色 1E2761、白文字、Bold

### 数値フォーマット
- 金額: 3桁カンマ区切り、通貨単位はヘッダーに記載する
- パーセンテージ: 小数1桁で統一する
- 正の増減率には + を明記する
- N/A・空欄はemダッシュ（—）に統一する

### 比較表の評価表現（◎○△×の代替）
- テキスト + 色分けで表現する:
  - 最高評価: "0D7C3D"（緑）+ テキスト "完全自律" 等
  - 標準: "333333"（黒）
  - 限定的: "F59E0B"（オレンジ）
  - 未対応/劣位: "C62828"（赤）
- 推奨列: 列全体の背景を "F0F4FF"（薄い青）にする

### 列幅
- カテゴリ列（最左）: 全体の25-35%
- 均等割りは禁止（コンテンツの比率に基づいて設定せよ）

### 必須要素
- ソース表記: テーブル下部に出典を記載する
- 表タイトル: テーブル上部にタイトルを記載する

## 品質検証

- [ ] ◎○△× が含まれていないこと
- [ ] 縦罫線がないこと
- [ ] 数値が全て右揃えであること
- [ ] ヘッダーと合計行が視覚的に区別されていること
- [ ] ソース表記があること
- [ ] 推奨列（あれば）がハイライトされていること
- 画像化してユーザーに提示し、フィードバックを得ること
