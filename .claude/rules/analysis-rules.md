# 分析・リサーチの作成ルール

## データ収集基準
- 各分析エリアにつき最低3つのデータポイントを収集する
- データポイントには必ず以下を含める:
  - 数値（具体的な定量データ）
  - 出典（著者/機関名、年）
  - 比較基準（前年比、競合比、業界平均比）
  - So What（このデータから何が言えるか）
- 出典は最低5つの異なるソースを使用する
- 1次ソース（公式発表、調査レポート）を優先する

## 分析フレームワーク
- MECE（Mutually Exclusive, Collectively Exhaustive）を意識して分類する
- ピラミッド原則: 結論→根拠→詳細の順で構成する
- 各セクションに「反論・リスク」の視点を含める
- クロスエリア分析: 異なる分析領域間の因果関係を明示する

## 数値・データの取り扱い
- 推計値には「(推計)」「(予測)」を明記する
- 為替レートが関わる場合は換算基準を明示する
- パーセンテージは母数を明記する（例: 「開発者1000名中の76%」）
- 時系列データは基準年を統一する

## ケーススタディ
- 企業事例には以下を含める:
  - 企業名・規模・業種
  - 導入前の状況（Before）
  - 導入後の成果（After）
  - 定量的な効果指標
  - 成功要因または教訓

## 品質チェック
- 全データポイントに出典が付いていることを確認する
- 数値の整合性を検証する（合計値、前年比の計算等）
- 古いデータ（3年以上前）には注釈を付ける
- 矛盾するデータがある場合は両論併記する
- データ更新日時を明示する
