# プレゼンテーション（pptx）の作成ルール v2

このルールはSTEP 2（ビジュアル実装）のスライド生成に適用される。slide-master-plan.md の設計仕様に基づいて実装すること。

## 最重要: 絶対禁止（コード生成前に確認 — 違反は全成果物やり直し）

以下に1つでも違反した場合、そのスライドは不合格・全体やり直し:

1. タイトル下にアクセントラインを入れるな → 発見次第やり直し
2. 装飾的な円・楕円・幾何学模様を配置するな → 発見次第やり直し
3. ◎ ○ △ × をテーブルのセルに書くな → テキスト評価+色分けで代替せよ
4. Unicode bullet "•" を直接書くな → bullet: true を使え
5. スライド下半分（y > 3.5inch）を空白にするな → y+h >= 4.5inch を必ず確認せよ
6. 全セル格子罫線を使うな → border: {type:"none"} + bottom個別指定

❌ `slide.addShape(pres.shapes.LINE, {x:0.5, y:1.0, w:3, h:0, line:{color:"1E2761", width:2}});`
→ アクセントラインのコードを書いた時点でやり直し。
❌ `rows.push(["項目", "◎", "○", "△"]);`
→ ◎○△×をテーブルに書いた時点でやり直し。

失敗パターンの詳細は `templates/anti-patterns.md` を参照。

## レイアウト
- スライドサイズ: LAYOUT_16x9（10" x 5.625"）
- タイトルスライドを必ず最初に配置する
- 最終スライドに「まとめ」または「Next Steps」を入れる
- 1スライドあたりのテキスト量は150文字以内を目安とする
- 余白は上下左右0.5インチ以上を確保する

## 座標テンプレート（prompt-slide-v2.md準拠）
- タイトル: x=0.5, y=0.3, w=9.0, h=0.6, fontSize=20, Georgia Bold
- ボディエリア: y=1.1 〜 y=5.15（ソース表記）
- ソース表記: x=0.5, y=5.15, w=9.0, h=0.3, fontSize=8
- 全スライドの最下要素の (y + h) が 4.5 inch 以上であること
- チャートの h は 3.5 inch 以上
- テーブルの rowH の合計は 3.5 inch 以上

## 空白防止チェック（必須）
- 全スライドで最下要素の y+h を計算し 5.15 inch 以上を確認
- テキストボックスの h がコンテンツ量に対して過大でないか確認
- カード/ボックスの内部が空白になっていないか確認

## フォント階層
- 見出し: Georgia Bold
- 本文: Calibri Regular
- タイトルスライド: 28-36pt / アクションタイトル: 20pt / 本文: 14-16pt / キャプション: 10-11pt

## ビジュアル
- すべてのスライドにビジュアル要素（グラフ、図、アイコン）を含める
- テキストのみのスライドは構造化テキスト（カード形式）に変換する
- グラフにはタイトル・凡例・単位・So Whatを必ず表示する
- 同じレイアウトパターンを3スライド以上連続させない
- 棒グラフで特定バーだけ色を変える場合は凡例で理由を明記する

## 配色ルール（70-25-5）
- 70% 白ベース背景 / 25% メインカラー / 5% アクセントカラー
- WCAG AAコントラスト比を確保する

## PptxGenJS固有ルール

### shadow設定
- ファクトリ関数 `makeShadow()` で毎回新しいオブジェクトを生成する
- 同一オブジェクトを複数要素で共有するな（違反は成果物やり直し）

### カラー指定
- hexカラーに `#` を含めるな（ファイル破損する。違反は成果物やり直し）
- 8桁hexでopacityを表現するな → transparencyプロパティを使え

### 禁止パターン
- ROUNDED_RECTANGLEとRECTANGLEのアクセントバーを組み合わせるな（発見次第やり直し）
- 紫グラデーション背景は禁止（AI生成の典型パターン。発見次第やり直し）
- lineSpacingとbulletを併用するな → paraSpaceAfter を使え

## python-pptxパイプライン固有ルール
- python-pptx の Chart API（add_chart）を使うな（Mac PowerPointで修復エラーが発生する。違反は成果物やり直し）
- チャートは全てシェイプベース（_rect + _text_box）で描画する
- ゾーン名はレイアウトの返す名前と完全一致させる

## 品質検証（v2: ユーザー確認ループ）
1. コード生成 → 実行（node generate-slides.js）
2. ユーザーにPPTXファイルを提示し確認を求める
3. 自分だけで「問題なし」と判断しない
4. フィードバックを受けて修正 → 再実行
5. Phase 6.5 Visual Validationも併用する（Error 0件を確認）
